## commit message rule
- feat : 새로운 기능 추가
- fix :  버그 수정
- build : 빌드 관련 파일 수정 / 모듈 설치 or 삭제 시
- chore : 그 외 자잘한 수정 (기타 변경)
- docs : 문서 수정
- style : 코드 스타일 혹은 포맷 등
- refactor : 코드 리팩토링
- test : 테스트 코드
- perf : 성능 개선
